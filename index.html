RESPONSIVE REDESIGN
How can we identify the usability issues of an existing interface and design a responsive website to address them?
THE ORIGINAL SITE

The site I chose to examine and redesign is the home page of the online store for Red Balloon Bookshop. I chose to analyze this webpage because it is affiliated with a book shop that was my absolute favorite store as a child, but the interface itself has some clear issues and is not particularly intuitive.

IS IT USABLE?
We can evaluate the usability of a web page by using three key metrics: learnability, efficiency, and memorability.
  1. Learnability: How difficult is the system to learn? Can a new user accomplish basic tasks easily?
  2. Efficiency: Can experienced users perform tasks quickly? Is a high level of productivity possible once a user is familiar with the system?
  3. Memorability: How quickly can users reestablish proficiency in the system after a period of not using it?
Using these metrics, I analyzed the original web page and identified the following key issues.

MAIN PURPOSE-RELATED
  1. The page is generally lacking a clear hierarchical structure that would make it more readable and easy to navigate.
  2. For a new or casual user who doesn't have a particular book in mind, this page is a bit intimidating. We can improve this by providing some browsing functionality.
  3. The "Buy Indie Online" section's layout and size detracts from the primary goal of this page, which is to find and order a book.
EXTRANEOUS INFORMATION
  1. The social media links detract from more significant key action options in the header.
  2. There is so much information given for each title in the pre-order section that it's tough to just skim the section.
REDUNDANCY
  1. The store hours are located in both the header and the footer, and the header is already very crowded.
  2. There are three search bars which all have the same function. However, the fact that there are three of them makes it seem like they should be used for different things.
UNCLEAR STRUCTURE
  1. In the header, the purpose of the search bar and the shopping cart is unclear, since the online shop is only a small part of the overall website.
  2. In the footer, the link to shop on kobo is very unintuitive.
NON-RESPONSIVE
  1. When the width of the browser window is decreased, the name of the shop and logo get cut off.
EVALUATING ACCESSIBILITY
We can use the Web Accessibility Evaluation Tool to get a quick overview of possible accessibility concerns that a web page might have. After using this tool on the original web page, a number of issues were identified.

The majority of issues involved very low contrast. Although not every example of this seemed like a valid concern to me, the white text on the light blue header, as well as the very light grey links on the white background did lack sufficient contrast to be easily readable. Additionally, the tool identified numerous instances of missing or insufficient alternative text descriptions for images on the web page. Given that these tags are necessary for accessibility tools such as screen readers, I agree that this is a significant accessibility problem.
LOW-FIDELITY PROTOTYPES
After identifying all of the significant usability issues in the original web page, I created wireframes detailing a new design that addressed these issues. I created three wireframes: one for desktop, one for tablet, and one for phone.

STYLE GUIDE

Before transitioning into web development, I built a style guide that displays the main colors, typography, and interactive components of the web page. The guide helps to ensure that styles and interactive elements stay consistent throughout the site in order to keep the user experience coherent.

HIGH-FIDELITY PROTOTYPES
After deciding on key components and styles, I used Figma to create high-fidelity prototypes for the same three screen sizes as my low-fidelity wireframes. These prototypes contain annotations of key layout choices to make the designs easier to reproduce using HTML/CSS.
